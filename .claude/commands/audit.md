# /audit — 원장 건강검진

범위: `$ARGUMENTS` (비어 있으면 전체)

## 점검 항목
1. **모순(contested)** — `by-confidence.md`에서 contested 주장을 모아 사람이 판단하도록 보고.
2. **저신뢰(low)** — 출처 보강이 필요한 low 주장 목록. 단일 출처/모델 추정 표시.
3. **고아(orphan)** — 어떤 토픽 뷰에서도 인용되지 않는 주장, 또는 깨진 `[^clm-id]` 각주.
4. **인덱스 정합** — `by-topic`/`by-time`/`by-confidence`가 실제 주장과 일치하는지.
5. **시간 일관성** — `valid_from > valid_until`, supersede 체인 끊김, 겹치는 active 기간.
6. **출처 무결성** — 모든 `sources.ref`가 실제 `20-raw/` 파일을 가리키는지.
7. **갭** — 토픽 뷰에 각주 없는 단언, 또는 자주 묻는데 주장이 빈 주제.

## 출력
심각도별(🔴 깨짐 / 🟡 주의 / 🟢 정보)로 정리한 보고서.
가능한 자동 수정(인덱스 재생성, 각주 복구)은 제안 후 수행, 판단이 필요한 건(모순 해소)은 사람에게.

## 규칙
- audit는 주장을 삭제하지 않는다. 상태 변경(retracted 등)은 근거를 남긴다.
- 자동 수정과 사람 판단 필요 항목을 명확히 구분한다.
