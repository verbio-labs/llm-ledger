# /timeline — 시간축 지식 추적

대상: `$ARGUMENTS`

LLM Ledger의 차별점: 사실을 덮어쓰지 않고 승계하므로 **과거 상태를 복원**할 수 있다.

## 절차
1. `aliases.md`로 정본 토픽을 확정.
2. `by-time.md`에서 해당 토픽의 주장을 `valid_from` 순으로 회수 (superseded 포함).
3. 두 가지 뷰 중 요청에 맞게 출력:
   - **변천사(기본)**: 시간순으로 "언제 무엇이 사실이었고, 무엇이 무엇을 대체했는지".
     각 전환에 옛/새 주장 id와 출처를 표시.
   - **as-of 스냅샷**: 날짜가 주어지면 "그 시점에 active였던 주장만"으로 당시 상태를 재구성.
4. contested 구간은 양쪽 입장과 기간을 함께 표시.

## 출력 예
```
napoleon — 권력 지위 변천
1799-11-09 ~ 1804-05-18  제1통령        [clm-2026-0001]
1804-05-18 ~ 1814-04-11  황제 (즉위)     [clm-2026-0007] supersedes 0001
1815-03-20 ~ 1815-07-07  백일천하 복귀    [clm-2026-0012]
```

## 규칙
- superseded 주장도 반드시 포함한다 (이게 타임라인의 핵심).
- 출처와 신뢰도를 함께 노출한다.
