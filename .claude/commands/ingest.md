# /ingest — 소스 수집

입력: `$ARGUMENTS`

목표: 자료를 **있는 그대로** `10-inbox/`에 저장만 합니다. 주장 추출이나 위키화는 하지 않습니다
(그건 `/compile`의 일).

## 절차
1. 입력 유형 판별:
   - **URL** → 본문을 가져와(web fetch) 마크다운으로 저장. 가져오기 실패 시 사용자에게 알림.
   - **파일 경로** → `10-inbox/`로 복사 (PDF·txt·md 등 원형 유지).
   - **붙여넣은 텍스트** → 마크다운 파일로 저장.
2. 파일명: `{추정-토픽}-{소스-슬러그}.md`. 토픽이 불명확하면 사용자에게 한 번 확인.
3. frontmatter 헤더 부착:
   ```yaml
   ---
   source_url: {원본 URL 또는 "local"}
   ingested: {오늘 날짜}
   status: uncompiled
   ---
   ```
4. 저장 후 보고: 파일명, 추정 토픽, "compile 대기 중"임을 1줄로.

## 규칙
- inbox에 있으면 = 아직 미처리. 절대 이 단계에서 주장을 만들지 않는다.
- 원문을 보존한다. 요약·편집 금지.
- 여러 소스를 한 번에 주면 각각 저장한다.
